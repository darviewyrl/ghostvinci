# Ghostvinci Game Logic

เอกสารนี้อธิบาย logic ของเกม Ghostvinci แบบอ่านง่าย เพื่อให้คนในทีมเข้าใจว่าเกมทำงานยังไงตั้งแต่เริ่มจนจบ โดยอิงจากโค้ดจริงของโปรเจกต์

## เกมนี้คืออะไร

Ghostvinci เป็นเกมเดายูกิโลแบบเล่นคนเดียวกับ AI ผู้เล่นจะผลัดกันทายการ์ดฝั่งตรงข้าม โดยมีเป้าหมายคือเปิดไพ่ที่ซ่อนอยู่ให้ได้มากที่สุด และใช้จังหวะการทายให้คุ้มก่อนเวลาหมด

เกมมีการ์ดทั้งหมด 26 ใบ:
- ไพ่ดำ 12 ใบ: `0` ถึง `11`
- ไพ่ขาว 12 ใบ: `0` ถึง `11`
- โจ๊กเกอร์ดำ 1 ใบ
- โจ๊กเกอร์ขาว 1 ใบ

ไพ่ทุกใบมีสีและค่าของตัวเอง แต่โจ๊กเกอร์มีระบบพิเศษ เพราะไม่ได้มีตำแหน่งตายตัวในตอนเริ่มต้น

## เป้าหมายของเกม

ผู้เล่นต้องเดาไพ่ที่ซ่อนอยู่ในมือของคู่ต่อสู้ให้ถูก

ถ้าเดาถูก:
- ไพ่ใบนั้นจะถูกเปิด
- ผู้เล่นได้คะแนน
- เกมอาจเข้าสู่ช่วงตัดสินใจว่าจะเดาต่อหรือหยุดเทิร์น

ถ้าเดาผิด:
- เกมจะลงโทษตามสถานการณ์
- ถ้ามีการ์ดจั่วอยู่ จะต้องเอาการ์ดใบนั้นเข้ามาในมือหรือวาง Joker
- ถ้าไม่มีการ์ดจั่วแล้ว จะเข้าสู่โหมดลงโทษให้เปิดไพ่ของตัวเอง

## โครงหลักของข้อมูล

สิ่งสำคัญที่สุดในเกมนี้คือ state ที่เก็บอยู่ใน reducer

ค่าหลัก ๆ ได้แก่:
- `gamePhase` - ตอนนี้เกมอยู่เฟสไหน
- `deck` - กองไพ่ที่เหลือ
- `playerHand` - ไพ่ของผู้เล่น
- `aiHand` - ไพ่ของ AI
- `activePlayer` - ตอนนี้เป็นตาฝ่ายไหน
- `drawnTile` - ไพ่ที่เพิ่งจั่วขึ้นมา
- `lastGuess` - ผลของการทายครั้งล่าสุด
- `pendingPhase` - เฟสถัดไปที่รอให้ acknowledge ก่อน
- `consecutiveCorrectGuesses` - จำนวนทายถูกติดกัน
- `turnCount` - จำนวนรอบ
- `elapsedTime` - เวลาเล่นรวม
- `timeLeft` - เวลาที่เหลือในเทิร์นปัจจุบัน
- `scores` - คะแนนของผู้เล่นและ AI

## ลำดับเกมตั้งแต่เริ่มจนจบ

## 1. Lobby

ตอนเริ่มเกมจะอยู่ที่หน้า lobby ผู้เล่นยังไม่เริ่มเล่นจริง

ในจุดนี้ทำได้แค่:
- เลือกระดับความยากของ AI
- เลือกจำนวนไพ่ที่จะตัดออกจากเกม
- เปิดหน้าตั้งค่า
- ดูคะแนนสะสม

เมื่อกดเริ่มเกม ระบบจะสร้างสำรับใหม่และเข้าสู่การแจกไพ่

## 2. Dealing

ช่วงนี้เกมจะแจกไพ่สลับกันระหว่างผู้เล่นกับ AI ทีละใบ

สิ่งที่เกิดขึ้นคือ:
- สำรับถูก shuffle ก่อน
- ไพ่บางใบถูกตัดออกตามค่าที่ตั้งไว้
- ไพ่แต่ละใบถูกแจกสลับซ้ายขวา
- ทุกครั้งที่แจกจะมี log บอกว่าใครได้รับไพ่อะไร

พอแจกครบ เกมจะไปตรวจว่าใครมี Joker ที่ต้องวางเองหรือไม่

## 3. Joker setup

ถ้าผู้เล่นได้ Joker ตอนแจกไพ่ เกมจะไม่ปล่อยให้ Joker อยู่มั่ว ๆ

ผู้เล่นต้องเลือกตำแหน่งที่จะวาง Joker ด้วยตัวเอง

หลักการคือ:
- เลือกช่องแทรก Joker ในมือ
- ระบบจะคำนวณค่า `assignedValue` ให้ Joker เพื่อให้มันเรียงในตำแหน่งที่เลือก
- ไพ่ในมือจะถูก sort ใหม่
- ถ้ายังมี Joker ที่ต้องวางอีก เกมจะให้วางต่อ

ส่วน AI จะจัดการ Joker ของตัวเองอัตโนมัติ

## 4. Turn banner

หลังจบการจัดการ Joker หรือจบการเปลี่ยนเทิร์น เกมจะโชว์ banner สั้น ๆ

ช่วงนี้ไม่ได้ให้ผู้เล่นทำ action หลัก แต่เป็นช่วงเปลี่ยนจังหวะเพื่อให้เกมอ่านง่ายและไม่กระโดดเฟสเร็วเกินไป

จากนั้นเกมจะไปเฟสถัดไปอัตโนมัติ

## 5. Draw

ในเทิร์นของแต่ละฝ่าย เกมจะจั่วไพ่ใบใหม่ขึ้นมาก่อน

ผลของการจั่วคือ:
- ไพ่ใบนั้นถูกเก็บไว้ใน `drawnTile`
- ถ้าการ์ดใบที่จั่วเป็น Joker จะมี logic พิเศษต่อ
- ถ้าไม่ใช่ Joker จะใช้ใบนั้นเป็นไพ่รองรับในเทิร์น

จุดนี้สำคัญมาก เพราะไพ่ที่จั่วมาจะเปลี่ยนได้ทั้งการเดาและการลงโทษ

## 6. Guess

นี่คือเฟสหลักของเกม

ผู้เล่นต้องเลือกไพ่ใบหนึ่งในมือฝั่งตรงข้าม แล้วทาย 2 อย่าง:
- ค่าของไพ่
- สีของไพ่

ทายถูกถึงจะนับว่าถูกจริง

### ถ้าทายถูก

ระบบจะ:
- เปิดไพ่ใบนั้น
- เพิ่มคะแนน
- เก็บข้อมูลผลทายไว้ใน `lastGuess`
- เข้าสู่ `FEEDBACK`

หลังจากนั้นเกมจะตัดสินว่าเทิร์นนี้จะ:
- เดาต่อ
- หรือหยุดเทิร์น

### ถ้าทายผิด

ระบบจะ:
- บันทึกว่าเดาผิด
- ดูว่าเทิร์นนี้มี `drawnTile` หรือไม่

ถ้ามี `drawnTile`:
- ไพ่ที่จั่วจะถูกเปิด
- ถ้าเป็น Joker จะเข้าสู่โหมดวาง Joker
- ถ้าเป็นไพ่ธรรมดา จะถูกใส่เข้ามือให้อัตโนมัติ

ถ้าไม่มี `drawnTile`:
- เกมจะเข้าสู่โหมดลงโทษ
- ผู้เล่นต้องเลือกไพ่ของตัวเอง 1 ใบมาเปิด

## 7. Feedback

เฟสนี้เป็นเหมือนห้องพักกลางทาง

เหตุผลที่มีเฟสนี้คือเกมไม่อยากให้ผลลัพธ์ของการทายชนกับเฟสถัดไปในจังหวะเดียวกัน

พฤติกรรมของมันคือ:
- แสดงผลว่าเดาถูกหรือผิด
- รอช่วงเวลาสั้น ๆ
- จากนั้นค่อยย้ายไปเฟสถัดไปจริง

เฟสถัดไปจะถูกเก็บไว้ใน `pendingPhase`

เมื่อถึงเวลาแล้ว ระบบจะส่ง `ACKNOWLEDGE_FEEDBACK` เพื่อเอาเฟสนั้นมาใช้จริง

## 8. Decide

เฟสนี้เกิดหลังจากเดาถูก และใช้ตอนที่ผู้เล่นต้องตัดสินใจว่าจะเล่นต่อหรือพอแค่นี้ในเทิร์นนั้น

ผู้เล่นสามารถ:
- เดาต่อ
- หรือผ่านเทิร์น

AI ก็ใช้ logic เดียวกัน แต่ตัดสินใจตามระดับความยาก

## 9. Joker placement

ถ้าการ์ดที่จั่วมาคือ Joker เกมจะไม่ให้เอาไปวางทิ้ง ๆ

ผู้เล่นต้อง:
- เลือกตำแหน่งแทรกในมือ
- วาง Joker ลงตรงนั้น

ถ้าผู้เล่นไม่ทันเวลา:
- เกมจะวาง Joker ให้อัตโนมัติ

หลังจากวางเสร็จ เกมจะเปลี่ยนไปเป็นช่วง turn banner แล้วเข้าเทิร์นต่อไป

## 10. Penalty reveal

ถ้าเดาผิดแล้วไม่มีไพ่จั่วรองรับ เกมจะบังคับให้เปิดไพ่ของตัวเอง 1 ใบ

กรณีนี้เกิดเมื่อ deck หมดแล้ว

ผู้เล่นต้อง:
- เลือกไพ่ที่ยังไม่เปิด
- ไพ่ใบนั้นจะถูกเปิดทันที

ถ้าพอเปิดแล้วไพ่ของผู้เล่นถูกเปิดครบหมด:
- เกมจบ

ถ้ายังไม่ครบ:
- เกมกลับไป turn banner
- แล้วสลับตาไปอีกฝ่าย

## 11. Game over

เกมจะจบเมื่อฝ่ายใดฝ่ายหนึ่งเปิดไพ่ครบหมด

เมื่อเกมจบ:
- timers หยุด
- หน้าจอแสดงผลแพ้/ชนะ
- ผู้เล่นเลือกเล่นใหม่หรือออกไปหน้า lobby ได้

## วิธีคำนวณคะแนน

คะแนนหลักจะเกิดจากการเดาถูก

สำหรับผู้เล่น:
- คะแนนพื้นฐาน 10
- บวกโบนัสตามความเร็ว `floor(timeLeft / 2)`
- บวก combo bonus จากจำนวนทายถูกติดกัน
- ถ้าชนะเกมด้วยการทายครั้งนั้น จะได้โบนัสเพิ่มอีก 30

สำหรับ AI:
- เมื่อทายถูกจะได้คะแนนตาม logic ของเกม

## การจัดเรียงไพ่

การเรียงไพ่มีหลักง่าย ๆ:
- ค่าน้อยอยู่ซ้าย
- ค่ามากอยู่ขวา
- ถ้าค่าเท่ากัน ไพ่ดำจะอยู่ซ้ายกว่าไพ่ขาว

โจ๊กเกอร์ใช้ค่าแฝง `assignedValue` เพื่อให้มันแทรกอยู่ตรงตำแหน่งที่ถูกต้อง

## AI ทำงานยังไง

AI ไม่ได้เดาสุ่มล้วน ๆ แต่ดูจากข้อมูลที่มีอยู่ก่อน

มี 3 ระดับ:

### Easy
- เลือกเป้าหมายแบบสุ่ม
- เดาสุ่มจากตัวเลือกที่เป็นไปได้

### Medium
- หาไพ่ที่น่าจะเดาง่ายที่สุดก่อน
- เดาสุ่มจากตัวเลือกของไพ่ใบนั้น

### Hard
- เลือกเป้าหมายที่มีความเป็นไปได้แคบที่สุด
- ถ้าแน่ชัดมากจะเดาแม่น
- บางครั้งอาจพลาดแบบใกล้เคียงเพื่อให้ดูเป็นธรรมชาติ

## Audio logic

เกมมีเสียงประกอบหลายแบบ:
- BGM สำหรับเล่นพื้นหลัง
- SFX ตอนจั่วไพ่
- เสียง flip / click / transition
- เสียงเตือนช่วงท้ายเวลา
- เสียงถูก / ผิด / ชนะ / แพ้

ข้อสำคัญคือ browser อาจบล็อกเสียงจนกว่าผู้เล่นจะกดหรือแตะหน้าจอก่อน

ดังนั้นระบบเสียงจึงมีแนวคิดนี้:
- ถ้า autoplay ไม่ผ่าน จะยังไม่ถือว่าพัง
- พอผู้เล่นมี interaction จริง ระบบจะ unlock แล้วค่อย retry
- เสียงเตือนเวลาเหลือน้อยจะไม่เล่นซ้ำถี่เกินไป

## Persistence

เกมจำ state ไว้ใน `localStorage`

ผลคือ:
- รีเฟรชหน้าแล้วบาง state ยังกลับมาได้
- audio setting ก็จำได้

ดังนั้นถ้าเปลี่ยนโครง state ต้องคิดถึงของเก่าที่ผู้เล่นอาจบันทึกไว้ด้วย

## จุดที่ต้องระวังเวลาจะแก้ logic

1. อย่าให้เฟสทับกัน
2. อย่าให้ feedback กับ phase ถัดไปเกิดพร้อมกัน
3. ถ้าแก้ audio ต้องเช็ก autoplay และ mute ด้วย
4. ถ้าแก้ Joker ต้องเช็กทั้งกรณีวางจากมือและวางจาก drawnTile
5. ถ้าแก้ timeout ต้องเช็กว่าโหมดเดา, โหมดตัดสินใจ, และโหมดวาง Joker ยังทำงานถูก
6. ถ้าแก้ state ต้องเช็ก localStorage เดิมด้วย

## สิ่งที่ควรทดสอบ

ถ้าจะแก้เกมนี้ ควรทดสอบอย่างน้อย:
- เริ่มเกมใหม่ได้ถูกต้อง
- แจกไพ่ครบ
- Joker setup ยังวางได้
- ทายถูกแล้วเปิดไพ่และให้คะแนน
- ทายผิดแล้วเข้า penalty / joker placement ถูกต้อง
- เวลาใกล้หมดแล้วเสียงเตือนยังไม่ spam
- เทิร์น AI ยังเดินต่อได้
- ออกจากเกมและกลับ lobby ได้

## สรุปสั้น ๆ

logic ของ Ghostvinci มีแกนหลัก 3 อย่าง:
- เกมเดินด้วย `gamePhase`
- ผลลัพธ์แต่ละครั้งถูกบังคับให้ผ่าน `FEEDBACK` ก่อน
- ไพ่ Joker และเสียงเป็นส่วนที่ต้องระวังที่สุด

ถ้าจะแก้อะไร ให้คิดก่อนว่าเรื่องนั้นเป็น:
- logic ของ reducer
- logic ของ UI
- หรือ side effect อย่างเสียงและเวลา
